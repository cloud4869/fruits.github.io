# Project Overview
This project is a browser-based game playable on PC and mobile browsers.
The goal is to build a simple, fun, and lightweight casual game step by step.

# Game Concept
- Genre: Casual browser game
- Session length: 1–5 minutes
- Easy controls
- Easy to understand rules
- Focus on gameplay over graphics

# Tech Stack
- HTML
- CSS
- JavaScript (Vanilla JS)
- Canvas API

# Development Style
Start from a minimal playable prototype as quickly as possible.
Improve the game incrementally with small features.
Avoid overengineering.

# Coding Rules
- Keep code simple and beginner-friendly
- Add comments in Japanese
- Prefer readability over clever solutions
- Avoid unnecessary libraries or frameworks
- Separate HTML, CSS, and JS files after prototype stage

# Project Structure
- index.html : main game screen
- style.css : styles
- game.js : core game logic
- assets/ : images and sounds
- docs/ : notes and ideas

# Game Design Rules
- Always keep the game playable
- Add one feature at a time
- Maintain smooth performance
- Ensure mobile-friendly layout
- Do not introduce complex systems unless requested

# Instructions for Claude
- Respond in Japanese
- Explain briefly before major changes
- Ask confirmation before large refactoring
- Suggest better implementations when appropriate
- Keep explanations concise and practical

# Workflow (IMPORTANT)
Always propose the next step after completing a task.
Break tasks into small actionable steps.
Prefer iterative development over large implementations.

# Development Priority
1. Playable prototype
2. Core gameplay loop
3. UI improvements
4. Sound and polish
5. Optimization

# Safety Rules
- Do not add external dependencies unless necessary
- Do not rewrite the whole project without permission
- Preserve working features when updating code
